# Project Summary
O AutoHub é uma plataforma inovadora para gestão de intermediação de veículos, permitindo que administradores gerenciem ofertas, fornecedores e compradores de forma eficaz. A solução utiliza um frontend em React e um backend em nuvem, integrando serviços com inteligência artificial para interações seguras. Recentemente, a plataforma implementou melhorias significativas, como a categorização automática de novos compradores na categoria "Geral", aumentando a eficiência na gestão de dados.

# Project Module Description
## Módulos Funcionais
- **Auth**: Gerenciamento de login/logout e sessões.
- **Dashboard**: Exibição de estatísticas gerais e ofertas recentes.
- **Suppliers**: CRUD completo para fornecedores de veículos.
- **Buyers**: CRUD para compradores, agora com associação automática à categoria "Geral".
- **Categories**: Gerenciamento de categorias para agrupamento de compradores.
- **Offers**: Gerenciamento do ciclo de vida das ofertas, incluindo status de rascunho gerado pela IA.
- **Distribution**: Distribuição de ofertas para categorias de compradores, com envio de mensagens via WhatsApp.
- **Negotiations**: Acompanhamento do status de negociações, com envio de atualizações via WhatsApp.
- **WhatsApp Settings**: Configuração da integração com a API Evolution para envio de mensagens e gerenciamento de assistente IA.
- **WhatsApp Conversations**: Gerenciamento de conversas com fornecedores, incluindo análise de mensagens e geração de ofertas.
- **Admin Phones**: Gerenciamento de múltiplos números de administradores que recebem notificações sobre ofertas e mensagens de compradores.
- **AIHub**: Serviço que suporta a variável `OPENAI_API_KEY` como fallback para autenticação com a API OpenAI.
- **WhatsApp Service**: Atualizações para lidar com redirecionamentos da API Evolution, melhorando a detecção de erros e mensagens de retorno.
- **Offer Workflow**: Gerenciamento do fluxo de aprovação de ofertas, incluindo análise de mensagens e fotos de veículos, com opções de negociação para compradores.
- **Admin Ad Workflow**: Fluxo que permite a criação de anúncios diretamente via WhatsApp, incluindo a gestão de sessões de administradores.
- **Admin Chat Commands**: Gerenciamento do sistema via comandos no WhatsApp, incluindo gerenciamento de contatos, categorias e configurações.
- **Admin AI Chat**: Serviço que permite conversas naturais com a IA, respondendo a perguntas e fornecendo informações sobre o sistema.
- **Plate Lookup**: Serviço de consulta de placas utilizando a API ConsultarPlaca como provedor principal e BrasilAPI como fallback.
- **Offer Media**: Armazenamento separado para imagens de ofertas, substituindo strings JSON na tabela de ofertas.
- **WhatsApp Events**: Armazenamento de eventos brutos de webhook para auditoria e depuração.
- **FIPE Lookup**: Funcionalidade para consulta de preços de veículos via API FIPE, permitindo consultas por placa ou código FIPE, com registro de logs de auditoria.
- **FIPE Command**: Comando no WhatsApp para consulta de preços FIPE, atualizado para aceitar placas, utilizando o fluxo de consulta da API ConsultarPlaca.
- **WhatsApp Admin Sessions**: Gerencia o estado de navegação administrativa via WhatsApp, agora com a adição de uma chave primária no modelo.

# File Description Inventory
- **backend/**: Lógica do servidor, incluindo autenticação, gestão de dados e rotas da API, agora com serviços e rotas para integração com WhatsApp e suporte ao modelo de IA OpenAI.
- **frontend/**: Interface do usuário, com componentes, páginas e lógica de interação, incluindo novas páginas para configuração do WhatsApp e gerenciamento de números de administradores, além da nova página de consulta de placas e consulta FIPE.
- **aihub.py**: Implementa o serviço de IA, agora com suporte à variável `OPENAI_API_KEY` para autenticação direta com a API OpenAI.
- **whatsapp.py**: Atualizado para processar corretamente mídias recebidas do WhatsApp, agora utilizando o campo `media` e incluindo `mimetype`.
- **fipe_lookup.py**: Serviço que permite consultas de preços de veículos usando a API FIPE, com registro de logs de consulta.
- **fipe_consultation_logs.py**: Modelo que rastreia cada consulta à FIPE para fins de auditoria.
- **whatsapp_admin_sessions.py**: Modelo atualizado para gerenciar o estado de navegação administrativa via WhatsApp, agora com chave primária.
- **admin_chat_commands.py**: Gerencia os comandos do chat do WhatsApp, agora incluindo a lógica para adicionar compradores automaticamente à categoria "Geral".
- **plate_lookup.py**: Serviço de consulta de placas utilizando a API ConsultarPlaca como provedor principal e BrasilAPI como fallback.
- **formatting.py**: Utilitários de formatação compartilhados para o sistema AutoHub.
- **consultarplaca.py**: Cliente API centralizado para ConsultarPlaca.

# Technology Stack
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, Shadcn/ui
- **Backend**: Atoms Cloud (Autenticação, Banco de Dados, Armazenamento de Objetos)
- **Banco de Dados**: PostgreSQL via Atoms Cloud
- **Autenticação**: Integrada ao Atoms Cloud
- **Armazenamento**: Object Storage para imagens de veículos
- **Integração**: API Evolution para envio de mensagens WhatsApp, API ConsultarPlaca para consulta de placas, API FIPE para consulta de preços.

# Usage
Para instalar as dependências e iniciar o projeto, siga os passos:
1. Navegue até o diretório do frontend e execute:
   ```
   pnpm install
   ```
2. Para construir o projeto, execute:
   ```
   pnpm run build
   ```
3. Para iniciar a aplicação, utilize o comando apropriado para seu ambiente de desenvolvimento.
4. Para utilizar as funcionalidades de IA, adicione a chave `OPENAI_API_KEY` nas configurações do projeto.
5. Para utilizar a consulta de placas, configure as credenciais `CONSULTARPLACA_EMAIL` e `CONSULTARPLACA_API_KEY` no Secret Manager.
6. Para utilizar a consulta FIPE, configure a chave `APIFIPE_TOKEN` nas configurações do projeto.
